Registration & Authentication Flow

before after

In addition to a redesign, I also reworked the registration flow. Previously, new users weren't required to set a password upon registration. This caused a lot of confusion if they returned to the site without ever setting their password and kept the call center busy with password help. I got rid of the unnecessary confirm email field since it's been shown that most users simply copy and paste their address from the original email field. I also included a button allowing users to show and hide their password on both registration and login in order to reduce errors and increase conversions.

Account Switcher UI

The account switcher feature is mainly used in-house to let employees switch between multiple accounts to help clients set up their events. The original account switcher was designed using the Kendo UI framework. Kendo was programatically generating the content each time the modal opened, forcing the browser to recalculate layout and the script to run slowly.

I included the basic layout in the view, so the script had less HTML to generate. I also stripped out Kendo from the feature and replaced it with a lighter script. The new account switcher takes one to two seconds less to load, saving the support team much frustration and minutes each month.
